02

Why Challengers

챌린저스를 선택한 이유

챌린저스를 처음 만난 건 아침 기상 챌린지 참여자로서였습니다. 매일 아침 앱에 인증 사진을 올려야 한다는 단순한 구조가 생각보다 강력하게 행동을 바꿨습니다. 앱이 사람을 바꾼다기보다, 바꾸고 싶은 사람의 의지를 구조가 뒷받침한다는 것을 직접 체감했습니다.

그리고 챌린저스가 습관 형성 앱에서 성과 연동형 커머스 마케팅 솔루션으로 피벗한 것을 알게 됐을 때, 그 전환이 자연스럽게 이해됐습니다.

요즘 마케팅의 본질은 맥락이라고 생각합니다. '많은 사람이 썼다'가 아니라 '저 사람이 이렇게 써봤더니 이렇더라'가 소비자를 움직입니다. 챌린저스는 그 맥락을 소비자 개인의 실제 사용 경험으로 만들어 줍니다. 브랜드의 메시지를 일방적으로 전달하는 것이 아니라, 소비자가 직접 써보고 인증하는 과정에서 자신만의 맥락을 만들어내는 것. 저는 이것이 챌린저스가 가진 가장 멋진 강점이라고 생각합니다.

저는 PM으로서 비슷한 구조를 설계한 경험이 있습니다. 기부금영수증 프로젝트에서 사용자의 행동 장벽을 데이터로 특정하고 제거했을 때 자가 발급 전환율이 47.5%까지 올라갔고, 비로그인 구조 확산으로 특정 콘텐츠에서 추가 결제가 10배 이상 증가했습니다. 사용자가 행동하기 쉬운 구조를 설계하면 비즈니스 지표가 직접 움직인다는 것을 경험으로 알고 있습니다.

챌린저스에서 저는 소비자가 자신만의 맥락을 만들어가는 경험을 더 정교하게 설계하는 데 기여하고 싶습니다.

01 배경 및 문제 정의

연말정산 시즌마다 CS 상담이 평소 대비 3~6배 폭증했습니다. CS 인력 증원, FAQ 보강 등의 대응을 반복했지만 이듬해에도 동일하게 반복됐습니다. 수요를 흡수하는 방식으로는 구조적 해결이 불가능하다고 판단하고, 사용자 행동 데이터로 원인을 특정하기로 했습니다.

GA 로그 분석

로그인 화면 이탈률 80% 이상 확인. 행동 장벽이 로그인 구조에 있음을 특정.

VOC 4년치 분석

전체 VOC의 80%가 단순 서류 발급 문의. 자가 처리 불가 구조 확인.

운영 맥락 분석

연말정산 즉시성 vs 계정 보유 전제. 구조와 맥락의 불일치 발견.

문제는 안내 부족이 아니라 사용자가 행동하기 어려운 구조 자체에 있었습니다. 연말정산이라는 즉시성이 중요한 맥락에서, 로그인이라는 전제 조건이 사용자의 흐름을 끊고 있었습니다.
02 가설

비로그인 간편 발급 구조를 제공하면, 행동 장벽이 사라지고 자가 발급이 증가해 관련 VOC 30% 이상 감소할 것이다.

  • 로그인 화면 이탈률 80% → 인증 구조 제거 시 이탈 대부분 해소 예측
  • VOC의 80%가 단순 발급 문의 → 자가 처리 가능해지면 CS 인입 직접 감소
  • 연말정산 즉시성 맥락에서 계정 보유 전제 구조는 본질적으로 맞지 않음
03 실행

구조 전환: 행동 장벽 제거

  • AS-IS: 회원가입 필수 → 로그인 → 나이스 실명인증
  • TO-BE: 내부 CRM 회원번호 + 이름 기반 간편 확인. 로그인 전제 조건 완전 제거

04 성과

  • 자가 발급 완료 14,439건 (서비스 유입 고객의 47.5%)

  • 수기 처리 → 자가 처리 구조 전환으로 운영 리스크 구조적 완화
  • 동일한 간편 인증 구조를 장기회원 리워드 서비스에 확장 적용

Project 02

마이페이지 운영 기준 수립

사용자 행동 데이터로 우선순위를 재정의해, 필요한 콘텐츠가 필요한 사용자에게 먼저 닿는 구조를 만든 프로젝트

콘텐츠 우선순위 데이터 기반 설계 운영 정책 플랫폼 고도화
기간2024.03 – 2024.05
역할PM 100% — 사용자 행동 데이터 기반 우선순위 설계, 부서별 요구사항 수집 및 협의, 운영 정책 수립
01 배경 및 문제 정의

서비스 확장에 따라 마이페이지 내 버튼·배너·CTA가 무분별하게 증가했습니다. 한 화면에 총 11개 기능이 노출되는 상황에서, 각기 다른 팀이 요청한 콘텐츠가 동일한 위계로 나열되고 있었습니다.

UI 구조적 사실

주요 기능과 부가 기능이 동일한 위계로 노출. 사용자 선택 부담이 과도하게 증가한 구조.

운영 맥락

부서별 CTA 추가 요청 반복. 공통 기준 부재로 운영 가이드도 복잡해짐.

플랫폼 성숙 단계

기능 중심 초기에서 사용자 목적 중심 확장기로 전환. 노출 기준 수립 필요.

진짜 문제는 기능 수가 아니라 우선순위의 부재였습니다. 어떤 콘텐츠를 누구에게 먼저 보여줄 것인가에 대한 기준이 없었고, 그 결과 사용자에게 선택 부담이 전가되고 있었습니다.
02 목표
  • 사용자 행동 데이터 기반으로 콘텐츠 우선순위 재정의
  • 전 부서가 납득할 수 있는 노출 기준 수립 → 이후 요청에도 동일 기준 적용
  • 리뉴얼 이후에도 다시 복잡해지지 않는 지속 가능한 운영 구조 설계
03 실행

부서별 요구사항 수집 및 충돌 조율

부서주요 요구사항
예산기획부서최근 구독 해지 급증 이슈를 서비스에서 방어하고 싶음
회원관리부서기존 상단 노출하던 결제/정산 정보를 유지하고 싶음
마케팅부서캠페인별 다양한 마케팅 서비스를 만들어서 제공하고 싶음
서비스부서만들어질 서비스의 효과성을 구체적으로 파악하고 싶음

운영 정책 문서화 → 협의 → 배포

  • GA·로그 데이터로 사용자 이벤트(클릭, 유입, 행동, 전환) 기준 기능 분석
  • 상단 노출 기준, 우선순위 조정 조건, 신규 기능 추가 프로세스 문서화
  • 정책 문서 기반 부서 간 합의 구조 형성
  • 배포 후 가이드라인 전사 공유 → 이후 요청에도 동일 기준 적용
04 성과
  • 전 부서 공통 운영 기준 수립으로 부서 간 요청 충돌 구조적 감소
  • 콘텐츠 노출 우선순위 재정의로 핵심 정보 접근성 향상 (내부 평가)
  • 운영 판단 속도 개선 및 의사결정 비용 절감
  • 후속 리뉴얼 프로젝트의 기준 구조로 활용
05 회고
이 프로젝트에서 배운 건 챌린저스가 매일 풀고 있는 문제와 닮아있습니다. 어떤 콘텐츠를 어떤 사용자에게 어떤 순서로 보여줄 것인가. 데이터가 충분하지 않은 상황에서도 사용자 행동 맥락을 종합해 우선순위를 정의하고, 그 기준이 이후 요청에도 지속적으로 적용될 수 있도록 구조화하는 것. 챌린저스에서 브랜드와 소비자 사이의 맥락을 더 정교하게 설계하는 데 이 경험을 직접 연결하고 싶습니다.
